Documentation

Component Inputs

Input Type Description
[settings] Object Table component configuration object, properties described below
[source] Array|DataSource If an array of objects is provided, it is automatically wrapped into a LocalDataSource.

Table Configuration

Property Type Default Description
Required Table Settings
columns Object undefined Table column settings, by default an empty object.
Example format:
columns: {{ '{' }} columnKey: {{ '{' }} title: 'Some Title' {{ '}' }} {{ '}' }}
Please note, columnKey must be the same as a key in data array objects.
Column Settings List of a column's settings
title string '' Column title
classHeader string '' CSS class that is applied to the column header cell.
classContent string '' CSS class that is applied to each column content cell. If you use a custom renderer, you can make use of this class by adding [ngClass]="cell.getColumn().classContent??''" to your HTML template.
class string '' Deprecated alias for classHeader.
width string '' Column width, example: '20px', '20%'
hide boolean false Whether to hide this column or not
isEditable boolean true Whether this column is editable or not. If this is false you can still enter a value when you add a new row. Use the isAddable setting to control the behavior when adding a new row. Note: the deprecated alias for this setting is editable.
isAddable boolean true Whether you can enter a value in this column when adding a new row.
type 'text'|'html'|'custom' 'text' If type is text then cell value will be inserted as text.
If type is html then cell value will be inserted as html.
If type is custom the renderComponent property must be defined.
sanitizer.bypassHtml boolean false Only for type html. Disables sanitizer. This allows inlining svg content. Security Note: Never use this feature, if the content can be controlled by a user.

Type must be custom in order for this to work.

Component Outputs/Events

Event Arguments Description
(afterGridInit) event: DataSet Fired once after the table grid is fully initialized. The argument describes the data set the grid has created from the data source after applying the table settings.
(rowSelect) event: RowSelectionEvent, consist of:
  • row: Object | null - the row or null when all rows are meant
  • data: Object | null - selected row data object
  • isSelected: boolean | null - the new selection state of 'data'
  • source: DataSource - table data source
  • selected: Array - the new array of selected elements
 Triggered when the selection state changes. The fields 'data' and 'isSelected' are null when 'row' is null.
(userRowSelect) event: RowSelectionEvent, consist of:
  • row: Object | null - the row or null when all rows are meant
  • data: Object | null - selected row data object
  • isSelected: boolean | null - the new selection state of 'data'
  • source: DataSource - table data source
  • selected: Array - the new array of selected elements
 Triggered only on a user click event. The fields 'data' and 'isSelected' are null when 'row' is null.
(rowHover) event: Row Triggered only on a user mouseover event.
(create) event: CreateEvent, consist of:
  • source: DataSource - table data source
 Triggered once a Create button clicked. Triggered only if table mode = external.
(createConfirm) event: CreateConfirmEvent, consist of:
  • newData: Object - data entered in a new row
  • source: DataSource - table data source
  • confirm: Deferred - Deferred object with resolve(newData: Object) and reject() methods.
 Triggered once a Create button clicked. Triggered only if table confirmCreate = true and mode = inline. Allows you to confirm changes before they are applied to the table data source.
(createCancel) event: CreateCancelEvent, consist of:
  • discardedData: Object - entered data that is discarded by the cancel
  • source: DataSource - table data source
 Triggered when the Cancel button in inline mode is clicked.
(edit) event: EditEvent, consist of:
  • row: Row - the row the edit operates on
  • data: Object - row data object
  • source: DataSource - table data source
 Triggered once an Edit button clicked on a row. Triggered only if table mode = external.
(editConfirm) event: EditConfirmEvent, consist of:
  • row: Row - the row the edit operates on
  • data: Object - original row data
  • newData: Object - edited data
  • source: DataSource - table data source
  • confirm: Deferred - Deferred object with resolve(newData: Object) and reject() methods.
 Triggered once a Save button clicked. Triggered only if table confirmSave = true and mode = inline. Allows you to confirm changes before they are applied to the table data source.
(editCancel) event: EditCancelEvent, consist of:
  • row: Row - the row the edit operates on
  • data: Object - original row data
  • discardedData: Object - edited data that is discarded by the cancel
  • source: DataSource - table data source
 Triggered when the Cancel button in inline mode is clicked.
(delete) event: DeleteEvent, consist of:
  • row: Row - the row that is supposed to be deleted
  • data: Object - row data object
  • source: DataSource - table data source
 Triggered once a Delete button clicked on a row. Triggered only if table mode = external.
(deleteConfirm) event: DeleteConfirmEvent, consist of:
  • row: Row - the row that is supposed to be deleted
  • data: Object - data object to delete
  • source: DataSource - table data source
  • confirm: Deferred - Deferred object with resolve() and reject() methods.
 Triggered once a Delete button clicked. Triggered only if table confirmDelete = true and mode = inline. Allows you to confirm changes before they are applied to the table data source.
(custom) event: CustomActionEvent, consist of:
  • action: string - the name of the registered custom action
  • row: Row - the row the custom action operates on
  • data: Object - the row data
  • source: DataSource - table data source
 Triggered once a custom action button clicked. There may be multiple custom actions that are distinguished by name.

Component Methods

Method Arguments Description
selectRow
  • index: number - the row to select
  • switchPageToSelectedRowPage: boolean - optional, override the respective table setting
 Selects a row programmatically. Currently only works reliably in 'single' selection mode.
getNotVisibleColumns none Returns an Array<IColumn> of all currently hidden columns.
toggleColumnVisibility columnId: string - the property name of the column to toggle. Inverts the hide flag on the specified column and reloads the table settings.
The other methods of this component should not be used by the user, because they are mostly internal event handlers.

Data Source Methods

Method Arguments Description
load
  • data: Array - data to load into the table
 Reload table with new data. For example if some data has received from the server.
prepend
  • element: Object - object to add
 Add a new element to the beginning of the table.
append
  • element: Object - object to add
 Add a new element to the end of the table. This also will switch current page to the last one.
add
  • element: Object - object to add
 Add a new element to the table.
remove
  • element: Object - object to remove
 Remove the element from the table.
update
  • element: Object - object to update
  • values: Object - object with new values
 Update the element in the table.
find
  • element: Object - object to find
 Find the element in the table.
getElements none Get elements for current sort, filter and page.
getFilteredAndSorted none Get all available elements for current sort and filter. Note, that for some data source implementation (like ServerDataSource) this
getAll none Get all data source elements.
updateSort
  • conf: Array - array of sort setting objects, object format is:
    • field - string - columnKey
    • direction - string|null - 'asc'|'desc'|null - sort direction
    • compare? - Function|null|undefined - optional custom compare function
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Updates the sorting configuration for the fields specified, leaving any other sorting configuration intact, example:
this.source.updateSort([{{ '{' }} field: 'id', direction: 'asc' {{ '}' }}]);
Note, that the latest added configuration will have the most significant sorting priority.
setSort
  • conf: Array - array of sort setting objects, object format is:
    • field - string - columnKey
    • direction - string|null - 'asc'|'desc'|null - sort direction
    • compare? - Function|null - optional custom compare function
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Replaces the entire sort configuration, removing anything that is not specified in the argument of this function. You should not use this function. Use updateSort, instead.
getSort none Returns an Array of sort setting objects (see setSort for the format).
setFilter
  • conf: Array - array of filter setting objects, object format is:
    • field - string - columnKey
    • search - string - search query
    • filter - Function|null - custom filter function
  • andOperator: boolean - how to process multiple filters (as AND or as OR), default = true (AND)
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Set table filters, example:
this.source.setFilter([{{ '{' }} field: 'id', search: 'foobar' {{ '}' }}, {{ '{' }} field: 'name', search: 'foobar' {{ '}' }}]);
addFilter
  • conf: Object - one filter object, format is:
    • field - string - columnKey
    • search - string - search query
    • filter - Function|null - custom filter function
  • andOperator: boolean - how to process multiple filters (as AND or as OR), default = true (AND)
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Set table filter for one column, example:
this.source.addFilter({{ '{' }} field: 'id', search: 'foobar' {{ '}' }});
getFilter none Returns an Object of the following form:
  • filters - Array - array of filter objects (see above)
  • andOperator - boolean - how the filters are combined (AND or OR), default = true (AND)
setPaging
  • page: number - page number
  • perPage: number - about per page
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Set table pagination settings
setPage
  • page: number - page number
  • doEmit: boolean - emit event (to refresh the table) or not, default = true. Set this to false if you want to prepare a configuration without refreshing the table (e.g. to avoid a HTTP request from a ServerDataSource).
Set table page
selectAllItems
  • checked: boolean - true: select all, false: unselect all
  • onlyFiltered: boolean - if set, only operate on the elements that match the current filter
Selects all (filtered) items
isEveryElementSelected
  • onlyFiltered: boolean - if set, only operate on the elements that match the current filter
  • reportEmptyAsFalse: boolean - if set, only operate on the elements that match the current filter
Returns true if and only if every (filtered) item is currently selected.
getSelectedItems none Returns an array of the currently selected elements.
reset
  • silent: boolean - if true - you have to additionally call `refresh` to reflect the changes
 Set data source to first page with empty filter and empty sort.
refresh none Refresh data in the data source. In most cases you won't need this method.
count none Returns the total number of elements with respect to the current filter. If you want to find the overall total number of elements, use getAll() and obtain the length - but keep in mind that this might be a costly operation.
empty none Empty the data source.
